Lesson 56 - Exceptions

This lesson introduces exceptions in Java, which are events that disrupt the normal flow of a program during execution.
They are commonly caused by errors such as invalid input, division by zero, or accessing unavailable resources.

*Topics Covered*
- What exceptions are and when they occur
- Using try blocks to wrap risky code
- Handling errors with catch blocks
- The role of the finally block
- Common exceptions like InputMismatchException and ArithmeticException

*Notes*
- The try block contains code that may cause an exception.
- The catch block handles specific exceptions and prevents the program from crashing.
- The finally block always executes, regardless of whether an exception occurs or not.
- Exception handling improves program stability and user experience.
- Multiple catch blocks can be used to handle different types of exceptions.
