Lesson 48 - Interfaces

This lesson introduces interfaces in Java, which act as blueprints for classes.
Interfaces define a set of abstract methods that implementing classes must provide, enabling a form of multiple inheritance-like behavior.

*Topics Covered*
- What an interface is and how it differs from a class
- Declaring interfaces using the interface keyword
- Implementing interfaces with the implements keyword
- Enforcing method implementation in classes
- Using multiple interfaces in a single class
- Polymorphic behavior through shared method contracts

*Notes*
- Interfaces contain abstract methods (no method body) that must be implemented by any class that uses them.
- A class can implement multiple interfaces, unlike inheritance with classes.
- The @Override annotation ensures that methods correctly implement interface methods.
- Interfaces promote flexibility, modularity, and loose coupling in object-oriented design.
- They are commonly used to define shared behaviors across unrelated classes.
