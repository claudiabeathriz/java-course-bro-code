Lesson 50 - Runtime Polymorphism

This lesson introduces runtime polymorphism in Java, where the method that gets executed is determined at runtime based on the actual object type.
It allows a superclass reference to call different implementations of a method depending on the object it refers to.

*Topics Covered*
- Concept of runtime polymorphism (dynamic method dispatch)
- Using superclass references to hold subclass objects
- Method overriding in subclasses
- Decision-making at runtime using user input
- Calling methods based on the actual object type

*Notes*
- The method that runs is based on the object’s actual type, not the reference type.
- This is achieved through method overriding in subclasses.
- Runtime polymorphism increases flexibility and allows programs to handle different behaviors dynamically.
- Commonly used in scenarios where the exact object type is not known until execution time.
